# App Configuration

export default ({ children }) => <DocsWrapper>{children}</DocsWrapper>

## Note

Below listed configurations can be changed during dockerbuild.

- default application configuration is applied from config/default.js.
- production.js overwrites mongo configuration.
- production.js overwrites minio configuration.
- production.js overwrites registry configurration.
- build.environment: openshift //must be set for OpenShift
- openshift.[ namespace | appPublicRoute ] //must be set for OpenShift
- smtp.enabled: true //must be set, else Bailo does not start the s2i build
- logging.file.enabled: false //set true to debug Bailo

## production.js

production.js can be overwritten during HELM deployment. See helm/bailo/templates/bailo/bailo.configmap.yaml

## **default.js**

This is stored at config/default.js
